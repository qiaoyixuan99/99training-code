## 一、程序形态决策

### 为什么不选原生 APP？

| 维度 | 原生 APP | 微信小程序 |
|------|----------|------------|
| 开发成本 | 双端开发，周期 6-12 月 | 一套代码，周期 2-3 月 |
| 推广难度 | 需下载安装，获客成本 ¥10-50/人 | 扫码即用，社交裂变零摩擦 |
| 更新速度 | 审核 1-3 天 | 审核快，热更新 |
| 用户留存 | 依赖推送权限 | 微信生态天然触达 |
| 支付转化 | 需接入支付 | 微信支付无缝衔接 |

### 为什么不选纯 H5 网页？

| 维度 | H5 网页 | 微信小程序 |
|------|---------|------------|
| 微信内体验 | 白名单限制，体验割裂 | 原生体验，性能好 |
| 分享能力 | 仅链接，无卡片 | 分享卡片 + 图文 + 视频 |
| 用户体系 | 需自建登录 | 微信一键授权 |
| 推送能力 | 无 | 服务通知 + 订阅消息 |
| 搜索流量 | SEO 竞争大 | 微信搜索 + 附近小程序 |

### ✅ 最终决策：**微信小程序** 为主，H5 为辅

```
         ┌──────────────────────┐
         │   微信小程序（主力）    │
         │  扫码即用 + 社交裂变   │
         └──────────┬───────────┘
                    │
         ┌──────────▼───────────┐
         │   H5 网页版（辅助）    │
         │  外部链接 + SEO 引流  │
         └──────────┬───────────┘
                    │
         ┌──────────▼───────────┐
         │   云后台 + AI引擎     │
         │   知识库 + 推荐系统   │
         └──────────────────────┘
```

**核心策略**：
- 小程序利用微信生态的**社交裂变**（学习打卡分享 → 好友点击 → 新用户）
- H5 利用**搜索引擎**和**外部链接**做流量补充
- 两套前端共享同一套云后台和 AI 引擎

## 二、产品定位

**一句话描述**：AI 驱动的股票交易学习助手，从零基础到实战进阶，个性化学习路径 + AI 导师陪练。

**目标用户**：
- 🟢 零基础小白（占比 60%）：需要系统入门教程
- 🟡 有经验投资者（占比 30%）：需要策略优化和深度学习
- 🔴 专业交易者（占比 10%）：需要量化工具和 AI 辅助分析

**核心差异化**：
- AI 根据用户水平动态调整学习内容
- 模拟交易 + AI 实时点评指导
- 知识图谱连接所有概念（点击任意术语 → 关联知识网络）

## 五、开发路线图（原始计划）

### Phase 1：MVP 核心（4-6 周）

| 周次 | 任务 | 交付物 |
|------|------|--------|
| W1 | 项目搭建 + 知识库结构化 | 基础框架 + 50 篇核心知识 |
| W2 | 首页 + 知识库浏览 + 搜索 | 可用知识库 |
| W3 | AI 对话接口 + 术语词典 | 智能问答 MVP |
| W4 | 用户体系 + 学习记录 | 登录 + 进度追踪 |
| W5 | 每日一题 + 章节测验 | 基础题库 |
| W6 | 测试 + 修复 + 上线 v1.0 | 可发布版本 |

### Phase 2：增长功能（4-6 周）
- 模拟交易 + AI 点评
- 学习打卡 + 分享裂变
- 能力雷达图 + 个性化推荐
- 社区广场

### Phase 3：变现功能（4 周）
- 会员体系（高级课程 / AI 深度分析）
- 策略商城
- 私教模式（AI 1v1）

### Phase 4：深度 AI（持续）
- 实盘信号辅助（需合规）
- 量化策略自动生成
- 跨市场分析
- H5 版适配

## 七、合规要点

- ⚠️ 不提供实盘买卖建议，仅提供教育和模拟
- ⚠️ AI 回答需添加免责声明
- ⚠️ 用户生成内容需要内容审核
- ⚠️ 小程序类目选择「教育-在线教育」
- ⚠️ 避免承诺收益、推荐具体股票
