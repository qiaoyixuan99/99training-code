## ADR-001: 前端选 Electron 而非 PyQt/PySide

- **日期**: 2026-06-09
- **状态**: ✅ 已确定
- **背景**: 需要跨平台桌面应用，Python 和 JS 均可做
- **选项**:
  - A: PySide6 纯 Python → UI 现代感不足，图表库少
  - B: Electron + React → 生态丰富，TradingView/Monaco 等现成组件
  - C: Tauri (Rust) → 包体小但生态不够成熟
- **决定**: 选 B (Electron + React)
- **理由**: 
  - TradingView Lightweight Charts 只有 JS 版本
  - Monaco Editor 只有 JS 版本
  - Ant Design 组件库成熟
  - 打包体积可接受（量化用户不介意几百MB）
- **代价**: 需要维护 Node.js + Python 两套环境

## ADR-002: 回测引擎自研 vs 使用 backtrader

- **日期**: 2026-06-09
- **状态**: ✅ 已确定
- **背景**: 需要 300只股票全历史 < 3分钟 的回测性能
- **选项**:
  - A: backtrader (成熟但慢，事件驱动)
  - B: zipline (已停维)
  - C: 自研向量化引擎
- **决定**: 选 C (自研)
- **理由**:
  - backtrader 事件驱动，300只需要 > 30分钟
  - 向量化回测 (NumPy + Numba) 比事件驱动快 100x
  - 可以参考 backtrader 的接口设计，保持兼容
  - 完全可控，后续可扩展
- **代价**: 开发工作量较大，需要充分测试

## ADR-003: 数据存储选 Parquet 而非 MySQL

- **日期**: 2026-06-09
- **状态**: ✅ 已确定
- **背景**: 历史行情数据量大（全A股日线 ~5GB），需要快速读取
- **选项**:
  - A: MySQL/PostgreSQL → 需要安装数据库服务
  - B: Parquet 文件 → 列式存储，直接读取
  - C: HDF5 → 生态不如 Parquet
- **决定**: 选 B (Parquet) + SQLite (元数据)
- **理由**:
  - 零配置，无需安装数据库服务
  - 列式存储：回测只读 OHLCV，跳过其他列
  - Snappy/Zstd 压缩省空间
  - PyArrow/Pandas 原生支持
  - 每只股票一个文件，并行回测时 IO 不冲突
- **代价**: 不支持并发写入（量化场景不需要）

## ADR-004: 数据源主选 AKShare

- **日期**: 2026-06-09
- **状态**: ✅ 已确定
- **背景**: 需要免费、稳定、覆盖全的A股数据源
- **选项**:
  - A: AKShare → 免费，覆盖广，更新活跃
  - B: Tushare Pro → 需要积分/付费
  - C: Baostock → 免费但更新慢
- **决定**: 选 A (AKShare 主力) + C (Baostock 补充)
- **理由**:
  - AKShare 完全免费，数据覆盖全（股票/指数/基金/期货）
  - Baostock 财务数据更全面，作补充
  - 设计 DataFetcher 适配器模式，随时切换数据源
- **代价**: AKShare 接口偶尔变动，需要跟进更新

## ADR-006: 缠论引擎架构设计

- **日期**: 2026-06-09
- **状态**: ✅ 已确定
- **背景**: 需要完整实现缠论分析管线
- **决定**:
  - 6模块分层架构: fractal → stroke → segment → center → buy_sell_point → chan_analyzer
  - 每层独立模块，可单独测试
  - 优先用线段构建中枢，降级可用笔构建
  - MACD背驰作为一二类买卖点核心判定依据
  - 5类异常覆盖：量能、缺口、背驰、假突破、异常振幅
- **理由**:
  - 分层对应缠论原文逻辑，便于验证每步正确性
  - 独立模块可在API层灵活组合（快速摘要 vs 完整分析）
  - 降级机制确保数据不足时仍能输出结果
- **代价**: 11步全流程计算量较大，长周期数据可能需优化

## ADR-007: 数据源双主备自动切换

- **日期**: 2026-06-09
- **状态**: ✅ 已确定
- **背景**: 单一数据源不稳定，需要容灾
- **决定**: Baostock(主) + AKShare(备) + 新浪(实时)
  - 日线及以上: Baostock 优先，失败自动回退 AKShare
  - 分时K线: 直接走 AKShare (Baostock 不支持)
  - 实时行情: 新浪财经 API
  - 全部带 TTL 缓存
- **理由**: Baostock 无需积分/AKShare 覆盖全/新浪实时免费
