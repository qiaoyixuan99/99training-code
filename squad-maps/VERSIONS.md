## v1 — 简单表单计算器

**功能：**
- 两个独立模块：武器测距 + 迫击炮计算
- 用户手动输入密位和目标高度
- 公式计算距离：`距离 = 高度 × 1000 ÷ 密位`
- 迫击炮密位通过 M252 81mm 射表线性插值

**特点：**
- 无需网络，纯本地运行
- 无地图，无交互
- 基础 CSS 样式

**缺点：**
- 完全依赖手动输入密位，不实用
- 无 Squad 地图数据
- 无瞄具模拟

## v2 — 交互地图版（含BUG）

**功能：**
- 20张 Squad 真实地图，通过 GitHub CDN 加载瓦片
- 鼠标交互：左键放置发射点、右键放置目标点
- 滚轮缩放（1-5级）、拖拽平移
- 实时计算距离、方位角、迫击炮密位
- 13种武器变体（不同国家/兵种组合）
- 6种瞄具分划绘制（ACOG、PSO-1、SUSAT、C79、Mil-Dot、红点）
- 弹道物理补偿（`calcHoldover` 函数，含枪口初速、重力、膛线高度）
- Ctrl+Z 撤销、M/T 切换放置模式
- 300m 主网格 + 100m 子网格 + 键盘编号

**已知BUG（故意保留）：**
1. **坐标转换NaN问题** --- `gameToScreen()` 使用 `const[p]=gameToPixel(...)` 解构，仅提取数组第一个元素，导致所有屏幕坐标为 NaN，标记/连线位置错误
2. **重复 mousedown 事件** --- 两个 `canvas.addEventListener("mousedown")` 同时触发，点击标记时会在原位置生成新标记，且可能干扰拖拽
3. **渲染时触发瓦片加载** --- `render()` 内部调用 `loadTile()`，每次加载完成触发 `requestAnimationFrame(render)`，造成无限循环和界面卡顿
4. **无本地瓦片** --- 完全依赖 GitHub CDN，首屏加载极慢
5. **复杂弹道模型** --- `calcHoldover` 使用真实物理公式，但与 Squad 游戏的简化弹道不匹配，补偿量不准确
6. **缺少 `weapon-ref-text` 元素** --- JS 引用了不存在的 DOM 元素
7. **100m 子网格无性能优化** --- 低缩放级别仍绘制所有子网格线，造成卡顿

## v3 — 最终修复版（当前使用）

**相对于 v2 的修复：**
1. **坐标转换修复** --- 改为 `const[px,py]=gameToPixel(gx,gy)`，正确解构
2. **合并事件处理器** --- 单一 mousedown 处理，按优先级：标记拖拽 → 平移 → 放置
3. **分离加载与渲染** --- `render()` 仅绘制已缓存瓦片，`batchLoadTiles()` 使用 `Promise.allSettled` 批量加载，不触发渲染循环
4. **本地瓦片优先** --- 首选本地 `tiles/` 目录（100张预下载图片），GitHub 兜底
5. **BDC 瞄具简化** --- 移除复杂弹道公式，直接匹配瞄具 BDC 刻线到目标距离，红点精确定位在对应刻线位置
6. **修复 DOM 引用** --- 补全所有缺失元素
7. **网格性能优化** --- 低缩放跳过子网格，超出5000/15000范围跳过绘制，标签仅在 zoom≥2 显示
8. **新增功能** --- F 居中、R 重置视图、S 交换位置、迫击炮/目标模式切换按钮

**v3 特性总结：**
- 本地瓦片秒开 + GitHub 自动兜底
- `ensureTiles()` 后台预加载新缩放级别瓦片
- BDC_POS 精确映射每种瞄具的刻线 Y 坐标
- `BDC_MARKS` + `getBDCMark()` 最近邻匹配
- 触摸屏支持（单指平移）
- ResizeObserver 自适应画布大小

## v4 — 零位控制瞄具版（最新）

**核心革新：瞄具红点位置由归零距离精确控制**

v4 参考 [squadmc.ende.pro](https://squadmc.ende.pro/) 的地图交互逻辑，并在此基础上实现了完整的弹道模拟 + 归零控制瞄具系统。

**v4 新增功能（对比 v3）：**

1. **真弹道物理引擎**
   - `calcHoldoverMils(dist, zeroDist, muzzleVel, scopeHeight)` — 完整弹道计算
   - 输入：目标距离、归零距离、枪口初速、瞄具高度
   - 输出：精确的 mil 补偿量
   - 物理模型：子弹从枪管出发（低于瞄具轴线 `scopeHeight`），受重力下坠，瞄具光轴与弹道在归零距离处相交

2. **归零控制红点位置（核心）**
   - 切换归零距离时，红点实时移动到新的瞄准位置
   - **归零 = 目标距离** → 红点在分划中心（0 mil 偏移）
   - **目标 > 归零** → 子弹下坠 → 红点下移（正 mil 补偿）
   - **目标 < 归零** → 子弹偏高 → 红点上移（负 mil 补偿）
   - 例如：目标 450m，归零 100m → 偏移 +3.2mil；切换归零到 400m → 偏移仅 +0.4mil

3. **squadmc 风格地图增强**
   - 射程环：以发射点为中心显示 200/400/600/800/1000/1250/1500m 范围圈
   - 50m 最小射程红线标记
   - 网格和射程环可独立开关
   - 弹道下坠厘米数显示
   - 更详细的面板信息

4. **更精确的瞄具分划**
   - BDC 刻线按真实 mil 角间距绘制（非等距排列）
   - ACOG: 100m=0mil, 200m=1.1mil, 300m=2.3mil, 400m=3.7mil...
   - Mil-Dot: 每点间距 1mil（30px/mil @ 6-10x）
   - 每种瞄具保留不同的 BDC_MARKS mil 间距定义

5. **武器弹道数据**
   - 8种武器（含精确射手步枪），每种含真实初速和瞄具安装高度
   - M110 SASS: mv=783m/s, scopeHeight=0.07m（更高瞄具安装）
   - 不同武器即使同瞄具也会显示不同红点位置

**v3 → v4 变化总结：**

| 特性 | v3 | v4 |
|------|----|----|
| 瞄具瞄准点 | BDC 最近邻匹配（静态） | 弹道物理 + 归零控制（动态） |
| 归零作用 | 仅信息显示 | **直接控制红点位置** |
| 弹道模型 | 无 | 完整物理（初速+重力+瞄具高度） |
| 瞄准补偿 | BDC 标记名 | 精确 mil 值（±0.1mil） |
| 射程环 | 无 | squadmc 风格多圈 |
| 图层控制 | 无 | 网格/射程环独立开关 |
| 武器数据 | 6种瞄具类型 | 8种武器（含弹道参数） |
| 弹道信息 | 无 | 实时下坠厘米数 |

**使用方式：**
1. 选择地图 → 放置发射点和目标点
2. 选择武器 → 调整归零距离
3. **观察瞄具中红点位置变化** — 切换归零，红点实时移动
4. 在游戏中将对应 BDC 刻线对准目标即可精确命中

## v6 — SMH 截屏 + 高度图版（最新）

**核心革新：参考 Squad Mortar Helper (SMH) 的截屏测距 + 高度图系统 + UI全面优化**

v6 参考 [squad-mortar-helper](https://github.com/WilliamVenner/squad-mortar-helper) 的设计理念，新增屏幕截取快速测距功能和高度图支持，并对UI进行了全面重构优化。

**v6 新增功能（对比 v5）：**

### 1. 屏幕截取系统（SMH 风格）
- **截取屏幕** — 使用 `getDisplayMedia()` API 捕获游戏画面
  - 选择窗口/屏幕 → 在游戏内打开地图(M键) → 按任意键定格
  - 自动加载截图到内置画布进行交互
- **粘贴截图** — 支持 `Ctrl+V` 从剪贴板直接粘贴游戏地图截图
  - 使用 PrintScreen 截图后一键粘贴
  - 支持剪贴板 API 读取图片
- **比例尺校准** — 点击地图上两个已知距离点完成标定
  - 例如：点击大网格线两端（300m）或子网格端点（100m）
  - 输入实际距离 → 自动计算像素-米转换比例
  - 校准后可精确测量任意两点距离
- **截图交互** — 在截图上左键=发射点、右键=目标点
  - 实时显示距离、方位角、迫击炮密位
  - 可视化标记点和连线

### 2. 高度图系统（Heightmap）
- **高度图加载** — 支持导入灰度PNG高度图文件
  - 文件存放于 `heightmaps/` 目录，命名为 `{map_key}_height.png`
  - 像素亮度(0-255)映射为实际海拔高度
  - 每张地图独立配置缩放比例和海拔范围（`HEIGHTMAP_META`）
- **实时海拔显示** — 鼠标悬停时在地图上显示当前坐标海拔
  - 工具栏显示高度图加载状态指示器（▲ HM）
  - 结果面板新增**高度差**行，显示发射点与目标之间的海拔差异
  - 上坡(蓝色)与下坡(红色)颜色区分
- **高度图来源** — 支持 Manual / Squad SDK / SMH Ripper 三种方式获取
  - `heightmaps/README.md` 包含完整使用说明
- **优雅降级** — 无高度图时所有功能正常运行，仅不显示海拔数据

### 3. UI 全面优化（SMH 风格）
- **标签页式侧边栏** — 三点标签切换不同功能区
  - 「计算」— 标记位置 + 射击参数 + 控制按钮
  - 「瞄具」— 武器选择 + 归零控制 + 瞄具视图
  - 「截屏」— 屏幕截取 + 粘贴 + 校准 + 快速测距
- **紧凑现代化设计** — 更小的间距和字号，最大化信息密度
  - 深色主题（GitHub 风格色板）
  - 工具栏显示高度图状态和坐标信息
  - 键盘快捷键提示精简
- **状态指示器** — 实时显示高度图加载状态、截图状态、校准进度
- **弹道下坠厘米数** — 射击参数面板新增弹道下坠实时显示
- **更好的移动端响应** — 优化小屏幕布局

### 4. 文件结构重组
- **`versions/`** 文件夹 — v1-v5 历史版本归档
- **`heightmaps/`** 文件夹 — 高度图存放目录（含使用说明 README.md）
- **`tiles/`** 文件夹 — 地图瓦片（保持不变）
- **`v6-smh-capture-heightmap.html`** — V6 主文件

**v6 技术要点：**
- Screen Capture API (`getDisplayMedia`) 无需浏览器扩展
- Clipboard API 支持从系统剪贴板读取图片
- Canvas 离屏渲染用于高度图像素采样
- 高度图缓存优化（ImageData 复用，避免重复 getImageData）
- 标签页按需渲染，切换时保持状态
- 全局 paste 事件监听，在截屏标签页自动触发
- 误差处理：截屏权限拒绝 → 引导使用粘贴方式
